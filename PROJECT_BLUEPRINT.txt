## 1. PROJECT OVERVIEW

This is a **real, production-ready** university portal project with full implementation of:
- Complete Django Ninja backend with system-aware models
- Configuration system supporting preset selection
- Docker Compose for easy deployment
- Support for British/Nigerian, American, and Polytechnic systems

### 1.1 System Presets

| Preset ID | Name | System Type | Style | Key Features |
|-----------|------|--------------|-------|--------------|
| `uni_british` | British/Nigerian University | University | British/Nigerian | NUC, JAMB, TETFund, CCMAS |
| `poly_nigerian` | Nigerian Polytechnic | Polytechnic | British/Nigerian | NBTE, ND/HND, SIWES |
| `uni_american` | American University | University | American | Majors/Minors, Liberal Arts |

## 3. CONFIGURATION SYSTEM

### 3.1 Configuration Presets

The system supports **3 preset configurations**:

```python
# Example: Selecting British/Nigerian University
config = get_config(
    style=AcademicStyle.BRITISH_NIGERIAN,
    system_type=SystemType.UNIVERSITY
)

# Example: Selecting Nigerian Polytechnic
config = get_config(
    style=AcademicStyle.BRITISH_NIGERIAN,
    system_type=SystemType.POLYTECHNIC
)

# Example: Selecting American University
config = get_config(
    style=AcademicStyle.AMERICAN,
    system_type=SystemType.UNIVERSITY
)
```

### 3.2 Environment Variables

```bash
# Required Configuration
ACADEMIC_STYLE=british_nigerian  # Options: british_nigerian, american
SYSTEM_TYPE=university           # Options: university, polytechnic

# Optional
UNIVERSITY_NAME=University of Nigeria
UNIVERSITY_SHORT=UNN
```

### 3.3 API Endpoints

```
GET  /api/config                    # Get current system configuration
GET  /api/config/presets            # Get all available presets
POST /api/config/set                # Set new configuration
GET  /api/system-info               # Get system information
```

## 4. DATABASE MODELS

### 4.1 Core Models (university app)

| Model | Description | Key Fields |
|-------|-------------|-------------|
| `University` | Institution | name, code, academic_system, system_type |
| `Faculty` | Faculty/College | name, code, dean |
| `Department` | Department | name, code, programme_types |
| `Programme` | Degree programme | name, code, programme_type, duration_years |
| `Session` | Academic session | session, is_current, tuition_fee |
| `Semester` | Semester | name, semester, start/end dates |

### 4.2 Student Models (students app)

| Model | Description | Key Fields |

| `Student` | Student profile | student_id, programme, level, status |
| `StudentDocuments` | Document storage | document_type, file, verified |
| `StudentStatusHistory` | Status tracking | old_status, new_status, reason |

### 4.3 Academic Models (academic app)

| Model | Description | Key Fields |
|-------|-------------|-------------|

| Model | Description | Key Fields |
|-------|-------------|-------------|

## 8. QUICK START

### 8.1 Using Docker

```bash
cd /workspace/university-portal

# Start with default (British/Nigerian University)
docker-compose up -d

# Or specify system
ACADEMIC_STYLE=british_nigerian SYSTEM_TYPE=university docker-compose up -d

# For American
ACADEMIC_STYLE=american SYSTEM_TYPE=university docker-compose up -d

# For Polytechnic
ACADEMIC_STYLE=british_nigerian SYSTEM_TYPE=polytechnic docker-compose up -d
```

### 8.2 Access Points

| Service | URL |
|---------|-----|
| API | http://localhost:8001 |
| API Docs | http://localhost:8001/api |
| Web | http://localhost:3000 |
| Centrifugo | http://localhost:8000 |
| PostgreSQL | localhost:5432 |
| Redis | localhost:6379 |

### 8.3 API Testing

```bash
# Get configuration
curl http://localhost:8001/api/config

# Get presets
curl http://localhost:8001/api/config/presets

# Set configuration
curl -X POST http://localhost:8001/api/config/set \
  -H "Content-Type: application/json" \
  -d '{"system_type":"university","academic_style":"british_nigerian","university_name":"University of Nigeria"}'
```
